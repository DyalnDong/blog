<p>万万没想到，这么短的时间，风水轮流转。记得 ChatGPT 刚出来的时候，使用很麻烦，就选择了 Claude 作为替代品，如今却很快调转了过来。听闻 A 社现在风控很严格，特地在美国选了个吉日回归，希望在后续的使用过程中能够有不杀之恩。</p>
<p>重新回归以后，发现此时的 Claude 已经不能称之为 Chatbox 了，确切点应该都可以叫做系统了。按我自己的理解方式，大概可以分成四层来说：最基础的对话，往上是文件和工具的协作，再往上是代码层面的 agent，最外面是外部工具连接和自动化，大概就是这样的一个模式。</p>
<p>基础层：对话</p>
<p>对话功能上都大差不差。但在体感上，值得讲的是记忆功能，现在 Claude 的记忆功思路上有点像 OpenClaw 的 Soul.md 了。用过 OpenClaw 应该知道，它有一个核心配置文件叫 Soul.md，你在里面写好这个 agent 的人设、规则、偏好，它之后所有的行为都会基于这个文件来走。</p>
<p>Claude 现在的记忆机制做的事情很类似，在对话过程中，有时候可以看到它自己去主动压缩对话，形成这个 Soul.md 文件。和 OpenClaw 区别是不需要你自己去写配置文件、维护 markdown，而是在正常对话过程中自然积累，相当于把 SOUL.md 的能力做成了一个对普通用户友好的版本，它可以自动去了解你的工作方式。</p>
<p>其他基础功能——Projects、web search、deep research、文件上传和生成—— 这些 ChatGPT 也都有，区别上就是回复风格的不同了，gpt 是个啰嗦的老太太，Gemini 是个佞臣，你做什么都有道理。Claude 人狠话不多，能一句话完成坚决不浪费笔墨，坚持它对的，就一定是你错，很桀骜不驯。</p>
<p>协作：Cowork</p>
<p>这个功能其实出了很久了，没用的时候感觉可以没有，但一旦用了后，会感觉好爽，为什么没有早点来用。和 Chat 最大的区别是对于工具的调用能力和工具的深度渗入。</p>
<p>比如Chat 里你得把东西上传给它，Cowork 里它可以直接来找你电脑上的东西。它跑在一个隔离的虚拟环境里，能访问你的本地文件系统，也能通过连接器接入外部服务——Google Drive、Slack、Gmail、Google Calendar 这些，相当于 OpenClaw 的变体了，从省心层面还是很有意义的。</p>
<p>但最最惊艳的，还是 Cowork 用插件的形式嵌入了 Excel、PowerPoint 和 Chrome。尤其是打工仔的使用场景下，我们不处理工作，只是工作的中转站。以做分析最常用的 Excel 场景来说，Claude 在 Excel 中构建了一个侧边栏，打开数据源，直接让它来帮你完成数据分析报表，实测几十万行的数据生成操作毫无障碍，堪比专业级的分析表，赏心悦目。PowerPoint 很久没做过了，但和 Excel 类似，应该都大差不差。</p>
<p>另外一个是浏览器插件，之前在云上部署网站的时候，很依赖这个功能，主要是用 Comet 来帮忙做，一个月也差不多得 20 美金左右。但现在有这个插件，基本上 Comet 就废了。一模一样的功能，还有你完整的记忆，对你的了解，以及强大的模型能力，有什么理由能拒绝呢。</p>
<p>还有个就是比较新的能力叫 Computer Use，其实就还是 OpenClaw 的变体了，能够直接控制你的鼠标、键盘、浏览器，模仿人在电脑上的操作，很适用那些没有外部连接可以用的场景，通过截屏来"看"你的屏幕，然后决定点哪里、输入什么。</p>
<p>远程：Dispatch</p>
<p>这个也是推动我转向 Claude 一个原因了。前两个月经常不是在给 OpencClaw 修 Bug，就是准备在修。每次都要打开电脑，而 Dispatch 就是提供了不用开电脑远程修复的一个功能，你能够在手机上给 Claude 派活，它直接在你的电脑上执行。</p>
<p>但也有个前提，就是电脑要保持开启状态，Claude 桌面端在运行。然后就可以用手机上的 Claude app，在 Dispatch 入口里下指令——比如"帮我把桌面上那个 Excel 整理一下"或者"扫一下邮件，把今天需要回复的列出来"——它就在你电脑上执行，做完了把结果发回手机。</p>
<p>配合 Computer Use 一起用的时候更有意思，人不在电脑前的时候，Claude 可以在你的电脑上打开应用、操作文件、跑完整个流程。比如你在通勤路上想到一件事，掏出手机说一句，等到了办公室，它已经做完了。</p>
<p>代码：Claude Code</p>
<p>这个属实是生产力工具了，但好像也是容易封号的重灾区。Claude Code 是跑在 Terminal 里面，虽说是给写代码的人用的，但其实并不一定非要写代码才能用，用途和 Chat 一样广泛。</p>
<p>Claude Code 本质上是一个 agent，你可以给它一个任务，它自己去读文件、写代码、跑程序、看报错、再改，直到跑通。平时一些重任务的时候，比如想搭建一个 Dashboard，或者做一些自动化的脚本之类，通常就直接丢到里面去跑，结果的完成度很高。</p>
<p>另外有个功能也很好，叫 Plan Mode，它的用处是：很多时候你想要一个产品，但头脑里并没有明确的概念想要什么。进了 Plan Mode 之后，它就会一步步问你问题："你这个功能是要给谁用的？""要不要支持 XX 场景？""这里有两种实现方式，你更倾向哪种？" 问到差不多了，它会给你一份 Markdown 格式的方案，列清楚要改哪些文件、每一步做什么、有什么 tradeoff。你看一遍，觉得没问题再让它执行；觉得有地方不对，可以直接在方案里改，或者继续跟它讨论。</p>
<p>这个流程的好处，就是相当于给你配了个顾问，顾问来一起完善成产品。直接给 agent 下一个模糊的指令让它跑，它可能跑出一个方向完全不对的东西，改起来比重做还累。用它的话，你不需要一开始就想得很清楚，它会帮你把模糊的想法问成一份具体的方案，起步成本更低了。</p>
<p>费用和其他</p>
<p>使用 Claude 的话，费用应该都不是问题了，如果交保护费能够避免风控的话，相信很多人是愿意付这个保护费的。看到的一种说法是，可以先注册使用免费版，免费版养养号，然后开 20 美金的版本，20 美金不够用了，触发过几次限额，再升级 100 美金或者 200 美金的版本。不要一次到位，风险成本很高。</p>
<p>另外注册的话，现在新注册是需要验证国外的手机了，注册好之后，按照上面的养养号，真要付费订阅的话，推荐用 App Store 直接绑卡支付，最好是外卡，GC 之类的风险非常高。现在也要说可能会实人 ID 验证，不过到时候就再说吧。</p>
<p>PS：另外还有个推动转变的理由，是哪里看到的一句话，你看美国政府都在使用的工具，还有什么理由不用呢。</p>
